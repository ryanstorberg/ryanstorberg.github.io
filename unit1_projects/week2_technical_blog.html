May 25Relative, Absolute, and Fixed Positioning

There are four different element position types available in CSS, static, relative, absolute, and fixed. Each elements default position type is static, which positions the element in the regular flow of CSS, stacked in order if it's a block element, lined up in order if it's an inline element.

position: relative. When positioning an element relatively, you're positioning it relative to itself, while also setting the stage for allowing greater control over the absolute positioning declaration. Also note that when an element is set to relative and its position moved, its previously occupied space remains a part of the natural flow of the elements, and the void remains unfilled.

position: absolute. An absolutely positioned element is positioned relative to either its nearest relatively positioned parent element, or the HTML document itself, if no relatively positioned parent element can be found. Unlike the relatively positioned element, the absolutely positioned element is taken out of the natural flow of the elements, and its void is filled by the next element in the HTML document.

position: fixed. Fixed elements are positioned relative to the display of the device that is viewing the webpage. They don't move from their initial position, even when the page is scrolled. It is also taken out of the natural flow of the elements, just like absolutely positioned elements.
